Importing W3C Tests into Webkit

Context

Where are we coming from?

  • Members of the CSSWG
  • Active with the W3C (Testing Task Forces, TestTWF)
  • Working on a team advancing a W3C Spec
  • The team is also implementing it in Webkit
  • Writing new tests to contribute to both Webkit & W3C
  • Continuing the dialog start here on this topic last year

We're writing new tests that should (will) live in two (or more) repos and we don't want to create a situation where (nearly) identical tests need to be maintained in two (or more) different places.

Basically, we want to do the right thing.

Hence,

Bug #111513

See #8:

http://trac.webkit.org/wiki/ImportingThirdPartyTests

What it does

(High Level)

  • Imports everything into LayoutTests/csswg
  • Only imports approved or submitted tests
  • Only imports automatable tests (reftests, testharness.js tests)
  • Converts files to work in Webkit
    • Refest filenaming convention
    • Paths to testharness.js
  • Adds vendor prefixes to CSS properties that require them
  • Logs the import
